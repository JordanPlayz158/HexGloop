      初入咒法

      研究这种魔法的人士会用法杖在空中绘制奇异的图案来施放他们口中的“咒术”，他们也会让强大的魔法物品替他们施法。我应该也可以这样做？

      “凝浆”至极的变化

      我最近发现世界上出现了某些“凝浆”极了的变化。

      例如，可视化图案变得更简单了。我甚至只需听见类似“\<southeast,deaqq>”的东西就能将其完美可视化为<southeast,deaqq>。此外，我还能将鼠标悬停其上以获得更为清晰的视图，还可点击它们以复制。


      我还发现翻笔记变得更顺畅了。按下打开Iota轮就能可视化法术书，并通过向下滚动法术书和向上滚动法术书快速翻页。

      到目前为止最奇怪的变化则是咒术的感觉不同了，甚至有种它们顺着我的骨头振动的感受。也许足够敏感的扫描器能发现其中区别？


      我还注意到最近施法时对照笔记更轻松了，按下打开咒术笔记就能让咒术笔记飞到面前，并打开上次翻到的那页。

      而书自身似乎……更平易友好了。打开到带有图案的一页并与悬挂的卷轴相碰，它就会将图案复制到卷轴上。甚至还会根据我用哪一只手持书决定复制哪一个图案。


      我还发现如果想让法杖更多姿多彩，可以为法杖覆上染料。

      咒术工具

      咒术大体上能替代剑和其他工具，但我必须承认工具本身带有某种天分。

      这些工具具有的咒术，再加上下界合金的感知力，便能兼顾两种优点。

      物品栏小猫

      我找到了我见过最可爱的猫猫！它似乎喜欢待在我的口袋里和物品玩耍。也许我能教它怎么玩耍更能帮到他人？

      我成功教会它施法了！给予它一个图案列表再轻抚它即可。这种操作和使用收纳袋相似得出奇。它们开始施法时的栈带有一些额外信息，首先是代表我是否拿起物品栏小猫的布尔值，然后是代表它所处或所看物品栏槽位的数。


      有部分图案只能由物品栏小猫绘制，这些图案大多都需要“槽位”（'slot'）参数。“槽位”并不指代某种实际的 iota 类型，而是可被物品栏小猫视作槽位的任意 iota：

      代表 GUI 中各槽位索引的非负数。箱子等容器中的槽位则由槽位索引 + 容器索引 * 1000 表示。

      施法者周围的物品实体。

      -2 代表施法时鼠标所选物品的槽位。

      -1000 * (物品栏与容器索引 + 1) 代表对应物品栏或容器中首个可用的目标槽位。

      环核

      截至目前，我只在法术环中使用过石板、促动石，以及导向石，但我最近发现的某些方块似乎在法术环中有独特的用处。我取了“环核”这个名称作为总括。后面几页主要用于介绍目前已发现的环核的功能。


      凝浆板岩箱子

      媒质流流过此类箱子时，法术环即可访问箱子的内容物，就像是访问施法者本人的物品一样。也许很适合巨型的方块工程计划。


      支座

      媒质流流过支座，且支座上物品存有图案（比如卷轴）时，法术环会将该图案加入环中用于施法。


      嵌镜支座

      这种支座和没有嵌镜的支座功能类似，但它同时会尝试向环嵌入非图案 iota。这种嵌入遵循的规则和往图案列表中嵌入 iota 相同。


      加速器

      到目前为止，法术环是所有施法方式中最慢的，这种方块应当能加快它们的速度。媒质流离开加速器时，其速度会加上抵达时速度的 200%。而每经过一格速度加成会减少 20%。单台加速器的运作需从促动石处抽取 2 个紫水晶粉，不过多次加速会使得消耗量迅速增长。在加速器之间留出区间应该能解决这种麻烦。
